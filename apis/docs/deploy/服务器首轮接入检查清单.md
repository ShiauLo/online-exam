# 服务器首轮接入检查清单

本文用于把 `online-exam` 的服务器首次接入做成一套可顺序执行的清单。

当前已确认的现实状态：

- 本地已有 SSH 客户端
- 本地已有默认密钥：`%USERPROFILE%\.ssh\id_rsa`
- 当前直接测试 `root@47.105.121.232` 时，服务器返回：
  - `Permission denied (publickey,gssapi-keyex,gssapi-with-mic,password)`

这说明：

- 网络基本可达
- 但服务器还没有接受本地这把 SSH 公钥
- 在修好 SSH 免密前，无法继续自动化部署脚本

## 1. 先确认本机公钥

在本机 PowerShell 执行：

```powershell
Get-Content "$HOME\.ssh\id_rsa.pub"
```

把整行公钥内容保存下来，后面要写入服务器的 `authorized_keys`。

## 2. 在服务器控制台写入 authorized_keys

如果你现在有服务器厂商控制台、宝塔终端或其他已登录入口，请在服务器上执行：

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
touch ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

然后把本机 `id_rsa.pub` 的整行内容追加进去：

```bash
cat >> ~/.ssh/authorized_keys
```

粘贴公钥后按：

```text
Ctrl + D
```

## 3. 检查 sshd 配置

在服务器执行：

```bash
grep -E 'PubkeyAuthentication|AuthorizedKeysFile|PermitRootLogin|PasswordAuthentication' /etc/ssh/sshd_config
```

建议至少满足：

```text
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
PermitRootLogin yes
```

如果改过配置，重启 sshd：

```bash
systemctl restart sshd
```

有些系统服务名可能是：

```bash
systemctl restart ssh
```

## 4. 本机验证 SSH 免密

本机执行：

```powershell
ssh root@47.105.121.232 "echo CONNECTED && whoami && hostname"
```

如果成功，说明第一步已经打通。

如果还失败，优先检查：

- 公钥是否真的写进了正确用户的 `authorized_keys`
- `~/.ssh` 和 `authorized_keys` 权限是否正确
- `PermitRootLogin` 是否允许
- 是否写错了用户，例如应该连 `ubuntu@...`、`ec2-user@...`

## 5. 服务器安装运行依赖

在服务器执行：

```bash
git --version
java -version
node -v
npm -v
```

如果缺少：

- Git
- JDK 21+
- Node.js 24.14.1+
- npm

先安装后再继续。

当前推荐安装边界：

- MySQL、Redis、RabbitMQ、Nacos：优先使用 Docker Compose
- Java / Node 微服务运行时：优先宿主机安装

这样能直接复用仓库里现有的 Git 拉取、定向构建、定向重启和日志脚本；
如果把应用也一起容器化，当前这轮部署链路会明显更绕。

## 6. 准备部署目录

建议固定目录：

```bash
mkdir -p /opt/online-exam
cd /opt/online-exam
```

如果仓库还没拉下来：

```bash
git clone git@github.com:ShiauLo/online-exam.git apis
mkdir -p /opt/online-exam/runtime-logs
mkdir -p /opt/online-exam/runtime-pids
```

如果仓库已经存在，只要确认目录为：

- Git 仓库根目录 `/opt/online-exam/apis`
- 工程目录 `/opt/online-exam/apis/apis`
- 运行时环境文件 `/opt/online-exam/.env`
- `/opt/online-exam/runtime-logs`
- `/opt/online-exam/runtime-pids`

建议同时检查：

```bash
sed -n '1,80p' /opt/online-exam/.env
```

至少应包含：

- `MYSQL_ROOT_PASSWORD`
- `REDIS_PASSWORD`
- `EXAM_NACOS_PASSWORD`

## 7. 检查服务器仓库 origin

在服务器执行：

```bash
cd /opt/online-exam/apis
git remote -v
```

必须是 SSH 地址：

```text
git@github.com:ShiauLo/online-exam.git
```

如果还是 HTTPS，改成：

```bash
git remote set-url origin git@github.com:ShiauLo/online-exam.git
```

## 8. 检查服务器到 GitHub 的 SSH

在服务器执行：

```bash
ssh -T git@github.com
```

如果失败，再检查：

- 服务器自己的 GitHub 私钥是否配置好
- `~/.ssh/config` 是否指向正确密钥
- GitHub 上是否录入了服务器公钥

## 9. 本机跑接入检查脚本

仓库已新增本地检查脚本：

- `docs/deploy/check-server-access.ps1`

执行方式：

```powershell
pwsh -File .\docs\deploy\check-server-access.ps1
```

它会检查：

- 本机到服务器的 SSH 免密
- 服务器仓库目录是否存在
- 服务器到 GitHub 的 SSH 是否可用

## 10. 接入完成后的第一条部署命令

当上面都通过后，本机执行：

```powershell
pwsh -File .\docs\deploy\deploy-server.ps1 -All
```

如果只想试一小部分服务：

```powershell
pwsh -File .\docs\deploy\deploy-server.ps1 -Services exam-account,exam-gateway
```

## 11. 当前推荐推进顺序

你现在最应该先做的是：

1. 把本机 `id_rsa.pub` 加进服务器 `authorized_keys`
2. 用本机 `ssh root@47.105.121.232 "echo CONNECTED"` 验证
3. 在服务器准备 Git 仓库 `/opt/online-exam/apis`，并确认工程位于 `/opt/online-exam/apis/apis`
4. 准备 `/opt/online-exam/.env` 并写入 MySQL、Redis、Nacos 真实密码
5. 跑 `pwsh -File .\docs\deploy\check-server-access.ps1`
6. 再跑 `pwsh -File .\docs\deploy\deploy-server.ps1 -All`
