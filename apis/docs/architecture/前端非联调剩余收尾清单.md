# 前端非联调剩余收尾清单

## 1. 目的

- 本清单只记录“不依赖后端联调”的前端剩余工作。
- 当前工程已具备独立运行、Mock 闭环、核心权限控制、关键测试与构建能力。
- 因此本清单中的事项均为“收口优化项”，不是阻塞交付项。

## 2. 当前结论

- 必须项：
  - 暂无新的前端独立功能缺口
- 主要剩余项：
  - `P2-3` 页面级代码复用尾项继续收口
  - 少量测试补强与文档同步

## 2.1 已完成的最近收口

- 弹窗表单栅格已抽为公共 `FormGrid`
- 学生答题页指标区已切换到共享 `MetricsOverview`
- 学生答题页题卡区已抽为公共 `ExamQuestionCard`

## 3. 剩余项明细

### 3.1 页面级复用继续收口

- 目标：
  - 将少量仍保留页面内私有实现的布局块继续抽成共享组件
- 当前范围：
  - 学生答题页的题卡与操作区结构
  - 各弹窗中的双列/三列表单栅格
  - 少量复杂操作区的说明块与状态标签区
- 验收口径：
  - 页面行为不变
  - 共享组件数量增加但抽象边界清晰
  - 不为了复用而复用，不引入额外理解成本

### 3.2 测试补强

- 目标：
  - 当页面结构进一步抽象后，补充受影响核心页回归测试
- 建议范围：
  - 学生答题页
  - 教师试题、试卷、考试页
  - 管理员账户页
  - 超级管理员角色权限页

### 3.3 文档同步

- 目标：
  - 每完成一个收口块，就把优先级文档与本清单同步到最新状态
- 验收口径：
  - 文档描述与当前代码一致
  - 能明确区分“已完成”“可选优化”“必须联调”

## 4. 不在本清单内的事项

- 真实接口字段适配
- 真实分页结构兼容
- 真实 Socket 事件对齐与重连握手恢复
- 真实错误码与空值兼容处理

- 以上事项统一归入 `P1-1 真实后端联调适配`

## 5. 建议推进顺序

1. 优先继续收口 `P2-3` 中复用价值明确的页面块
2. 每完成一个大块，立即执行增量测试与 `npm run build`
3. 非联调优化收口完成后，等待后端条件具备再切入 `P1-1`
