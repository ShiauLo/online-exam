# 线程 A 冻结口径回执（发给线程 B：exam-class / exam-system）

## 文档版本

- 版本：V1.0
- 修订时间：2026-04-03
- 修订人：线程 A

## 文档定位

- 设计目标：作为线程 A 对线程 B 前置依赖清单的正式回执，明确哪些共享口径已经冻结、哪些实现仍待线程 A 落仓，确保线程 B 可以先按统一标准继续推进 `exam-class` 与 `exam-system`。
- 适用范围：线程 B 首轮模块骨架、接口骨架、错误响应、鉴权接入、权限查询与系统配置相关接口的设计与实现。
- 当前边界：本文档冻结“共享口径”，不代表所有对应代码已经落仓；若文档口径与现有代码冲突，以本回执口径为准。

## 一、线程 A 正式冻结的共享口径

### 1. 统一响应结构

后续所有 Spring Boot 微服务统一响应字段固定为：

- `code`
- `msg`
- `data`
- `requestId`
- `timestamp`
- `errors`

统一要求：

- 成功响应默认 `code=200`、`msg=success`
- 参数错误统一 `400`
- 未登录 / token 无效 / 登录态失效统一 `401`
- 权限不足统一 `403`
- 资源不存在统一 `404`
- 业务状态冲突统一 `409`
- 服务器内部错误统一 `500`

补充说明：

- `errors` 仅在参数校验失败等需要返回明细时出现
- 所有错误响应也必须返回 `requestId` 与 `timestamp`

### 2. 鉴权边界与上下文透传

线程 A 冻结如下规则：

- 网关统一负责 token 合法性校验
- 业务服务默认不重复做 token 合法性校验
- 网关向业务服务透传以下请求头：
  - `X-User-Id`
  - `X-Role-Id`
  - `X-Request-Id`
- 外部请求链路追踪头统一使用：
  - `Request-Id`
- 过渡期允许 `Authorization` 原样继续透传，兼容当前仍在本地解析 token 的服务

`401` / `403` 规则：

- 网关与业务服务均使用统一响应结构包装
- `401` 用于未登录、token 无效、登录态失效
- `403` 用于已登录但无操作权限

### 3. 角色编码与 `roleId` 映射

线程 A 冻结如下口径：

- 对外接口返回角色编码采用小写：
  - `student`
  - `teacher`
  - `admin`
  - `super_admin`
  - `auditor`
  - `ops`
- 服务内部枚举与数据库角色编码采用大写：
  - `STUDENT`
  - `TEACHER`
  - `ADMIN`
  - `SUPER_ADMIN`
  - `AUDITOR`
  - `OPERATOR`

稳定映射如下：

1. `1 = SUPER_ADMIN`
2. `2 = ADMIN`
3. `3 = TEACHER`
4. `4 = STUDENT`
5. `5 = AUDITOR`
6. `6 = OPERATOR`

补充说明：

- `ops` 对外映射到内部 / 数据库的 `OPERATOR`
- 线程 B 在权限判断、菜单模板与审计字段中按此口径固定实现

### 4. 分页查询公共口径

线程 A 冻结如下规则：

- 分页字段统一使用：
  - `pageNum`
  - `pageSize`
- 字段类型统一为数值型
- 默认值：
  - `pageNum=1`
  - `pageSize=10`
- 边界校验：
  - `pageNum >= 1`
  - `1 <= pageSize <= 100`

### 5. 审计留痕最小字段

线程 B 涉及高危或管理操作时，最小审计字段统一按以下口径预留：

- `requestId`
- `operatorId`
- `operatorRole`
- `ip`
- `target`
- `action`
- `result`
- `approver`
- `timestamp`

## 二、线程 B 可直接采用的接入规则

### 1. 网关路由口径

线程 A 正式冻结如下服务命名与前缀：

- `/api/class/** -> lb://exam-class`
- `/api/system/** -> lb://exam-system`

首轮默认策略：

- 无白名单接口
- 全部受鉴权保护
- 错误响应按统一结构返回

### 2. 根工程接入口径

线程 B 可按如下目标结构接入：

- 新增模块：`exam-class`
- 新增模块：`exam-system`
- 聚合方式继续沿用当前 Maven 聚合工程模式
- 依赖接入继续沿用现有 Spring Boot 微服务模式

补充说明：

- 当前根工程 `pom.xml` 尚未实际纳入这两个模块
- 但线程 A 已冻结“允许纳入”的工程策略，线程 B 可按此设计模块结构

## 三、线程 B 依赖但线程 A 仍待落仓的实现项

以下内容口径已冻结，但代码仍待线程 A 后续统一落仓：

### 1. `exam-common`

- 下沉统一响应类型
- 下沉分页公共对象 / 校验模板
- 下沉鉴权头常量与 requestId 获取工具
- 下沉异常处理模板

### 2. `class/system` 错误码与异常

线程 A 将继续为线程 B 收敛首批共享异常与错误码，至少覆盖：

- 班级不存在
- 班级码无效
- 教师建班数量超限
- 学生入班数量超限
- 学生已在班级中
- 学生存在待审批申请
- 强制班级不可自主退班
- 教师无权操作非本人班级
- 删除班级前仍存在已加入学生
- 入班审批状态冲突
- 角色不存在
- 权限不存在
- 权限分配越权
- 配置项不存在
- 配置修改需要二次验证
- 日志导出需要审批
- 数据备份失败
- 数据恢复失败
- 备份 / 恢复状态冲突
- 临时授权已过期或无效

建议线程 B 在首轮实现中先按“共享异常占位 + 固定 HTTP 状态映射”设计，不自行扩散新公共异常。

### 3. `exam-gateway`

- 新增 `/api/class/**`
- 新增 `/api/system/**`
- 增加 `X-Role-Id`
- 增加 `X-Request-Id`
- 统一 `401 / 403` 包装逻辑

## 四、线程 B 即刻可继续推进的范围

在线程 A 尚未把上述实现全部落仓前，线程 B 仍可立即推进：

- `exam-class` 模块骨架
- `exam-system` 模块骨架
- Controller / Service / DTO / Exception 结构搭建
- 按本文档口径设计接口与错误响应
- 按本文档口径预留鉴权上下文读取方式
- 按本文档口径预留分页、审计与角色映射

线程 B 当前不应继续做的事：

- 在服务内再定义一套新的 `Result`
- 自行发散新的 Header 命名
- 自行改变 `401 / 403` 边界
- 自行定义与本文档冲突的角色编码或 `roleId` 映射

## 五、线程 B 首轮开工判定

线程 A 视以下条件为“口径已解锁，可按冻结标准开工”：

1. 统一响应结构已冻结
2. 鉴权透传边界已冻结
3. `/api/class/**` 与 `/api/system/**` 路由命名已冻结
4. 角色映射、分页口径、审计字段已冻结

补充说明：

- 上述是“设计口径解锁”，不是“全部代码实现完成”
- 若线程 B 后续新增确属公共层的 DTO / 枚举 / 异常，应继续回收给线程 A 统一收敛
