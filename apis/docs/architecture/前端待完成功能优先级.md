# 在线考试系统前端待完成功能优先级

## 1. 文档目的

- 本文档用于标记 `web/` 前端工程当前已完成能力、待完成能力、实施优先级与验收标准。
- 优先级排序遵循：
  - 先补影响权限正确性、业务安全性、角色边界的能力
  - 再补影响联调与验收闭环的能力
  - 最后补体验增强与性能优化

## 2. 当前已完成范围

- 独立 `Vue3 + TypeScript + Vite + Pinia + Vue Router + Element Plus` 工程已搭建完成。
- 六角色主页面、主路由、基础动态权限、登录刷新链路、错误页与个人中心已落地。
- 学生答题页已具备倒计时、自动保存、切屏上报、提交试卷等核心流程。
- `P0-1` 全站按钮权限、危险操作确认、关键表单校验已补齐到正式业务页。
- `P0-2` 问题流转实时通知已接入学生、教师、管理员、审计员、运维问题页。
- `P0-3` 关键业务验收测试已补齐到教师班级上限、教师暂停考试、成绩发布、双验证码恢复、审计员只读等场景。
- `P2-1` 正式业务页统一空状态、失败态与重试入口已完成收口。
- Mock 接口与基础单元测试已可运行，当前工程可通过：
  - `npm run test:run`
  - `npm run build`

## 3. 优先级总览

### P0：必须优先完成

#### P0-1 全站按钮权限与危险操作确认补齐

- 目标：
  - 将所有存在写操作的业务页统一接入按钮权限控制
  - 将删除、冻结、恢复、发布、终止、关闭、审批、转派、导出等操作统一接入二次确认
  - 补齐关键表单的前端必填校验和边界校验
- 当前状态：
  - 已完成
  - 正式业务页的写操作按钮已统一接入权限控制
  - 删除、冻结、恢复、发布、终止、关闭、审批、转派、导出等高风险操作已统一接入二次确认
  - 关键弹窗提交按钮与关键表单校验已完成收口
- 验收标准：
  - 无权限按钮不显示
  - 高风险操作全部二次确认
  - 关键表单不允许空提交或明显非法提交

#### P0-2 问题流转实时通知落地

- 目标：
  - 将 `issueNotify`、`processNotify` 事件真正接入学生、教师、管理员、审计员、运维的问题页
  - 在问题流转页实时刷新通知或时间线
- 当前状态：
  - 已完成
  - `issueNotify`、`processNotify` 已接入学生、教师、管理员、审计员、运维问题页
  - Mock 模式下已支持问题新建、接单、转派、关闭后的实时提示与列表刷新
- 验收标准：
  - 问题新建、接单、转派、关闭后，相关页面可收到实时提示或列表刷新

#### P0-3 关键业务验收测试补齐

- 目标：
  - 将目前“能加载页面”的测试扩展为“关键业务规则正确”的测试
- 重点补测：
  - 教师最多创建 1 个班级
  - 运维无恢复入口
  - 超级管理员双验证码恢复校验
  - 审计员页面只读
  - 关键按钮权限显隐
- 验收标准：
  - 新增测试稳定通过，且不依赖偶发等待
 - 当前状态：
  - 已完成
  - 已补充实时通知测试与关键业务验收测试
  - 当前前端测试已覆盖教师班级上限、教师暂停考试、教师发布成绩、超管双验证码恢复、运维无恢复入口、审计员只读、按钮权限显隐等场景

### P1：联调闭环优先

#### P1-1 真实后端联调适配

- 目标：
  - 将 `VITE_USE_MOCK=false` 场景下的接口字段、分页结构、错误处理、空值兼容调整到可联调状态
- 当前状态：
  - 前端运行时已补 `VITE_WS_BASE_URL`、`VITE_ISSUE_WS_BASE_URL` 与显式 `VITE_USE_MOCK` 开关
  - 当前默认本地开发环境已切到真实后端优先，mock 改为显式开启
  - `web + gateway + realtime + issue-notify` 启动链路已打通，当前主要剩余阻塞为本地数据库需同步最新 `docs/sql/mysql.sql`
  - 若数据库缺少 `exam_instance`、`exam_issue_record` 等新表，真实联调会直接返回缺表错误，而不是前端字段适配问题
- 验收标准：
  - 主要业务页在真实接口下可正常查询、提交、反馈错误

#### P1-2 实时连接稳定性增强

- 目标：
  - 增加断线重连提示、重连后的状态恢复、异常提示统一化
- 当前状态：
  - 前端侧首轮增强已完成
  - Mock Socket 已支持 `disconnect`、`reconnect`、`connect_error` 生命周期事件模拟
  - 学生、教师、管理员、审计员、运维问题页已统一接入断线提示、恢复提示与异常提示
  - 学生答题页已补齐断线提示、恢复中状态、重连后重新同步考试会话与倒计时恢复链路
  - 剩余工作集中在真实后端联调阶段，对齐服务端 Socket 事件命名、重连握手与状态恢复契约
- 验收标准：
  - 重连后倒计时、答题状态、问题通知可恢复
