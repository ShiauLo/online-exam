# 在线考试系统接口文档（V3.14 统一版）

## 文档版本

- 版本：V3.14
- 修订时间：2026-04-22
- 修订人：Codex

## 文档定位

- 设计目标：统一在线考试系统的接口路径、方法、参数、响应、权限校验和调用边界，作为后端开发、前后端联调和测试验收的唯一接口依据。
- 适用范围：本期确认的账户、班级、试题、试卷、考试、成绩、系统管理、问题申报和静态资源服务。
- 版本边界：本版仅覆盖本期实现，不包含多租户独立网关、第三方单点登录和离线考试客户端。
- 本期角色范围：`student`、`teacher`、`admin`、`super_admin`、`auditor`、`ops`。
- 非功能目标：满足统一鉴权、请求可追踪、双技术栈协同、生产安全和可测试性。
- 冲突处理原则：`角色权限以角色文档为准，接口定义以接口文档为准，页面交互以前端文档为准`。

## 一、技术栈与对外调用原则

| 微服务板块 | 核心技术栈 | 设计理由 | 对外职责 |
| --- | --- | --- | --- |
| 账户服务 | Spring Boot | 数据强一致、权限敏感 | 账户、认证、密码、登录日志 |
| 班级服务 | Spring Boot | 事务清晰、规则稳定 | 班级、入班、退班、导入导出 |
| 试题服务 | Spring Boot | 业务规则复杂、文件校验多 | 试题、分类、审核、批量导入 |

| 考试实时服务 | Node.js + Socket.io | 高并发低延迟 | 进入考试、自动保存、倒计时、切屏上报 |
| 成绩服务 | Spring Boot | 评分与审计要求高 | 评分、成绩明细、分析、申诉 |

| 资源服务 | Spring Boot | 统一复用本地资源目录与鉴权口径 | 图片、模板、导出文件下载 |

### 1.1 调用边界

1. 非实时业务接口统一经 HTTPS 网关对外暴露，前端调用路径统一为 `https://api.exam-system.com/api/...`。
2. 实时考试与实时通知接口统一通过 `wss://realtime.exam-system.com` 建立连接。
3. 前端不直接感知内部 Spring Boot 服务地址，非实时请求均通过网关转发。
4. Node.js 与 Spring Boot 的内部调用由服务间鉴权控制，不对前端暴露内部地址。

## 二、通用开发规范

### 2.1 请求与响应格式

1. 复杂列表查询统一使用 `POST /query`。
2. 文件下载类接口保留 `GET`，参数通过 Query String 传递。
3. 所有成功响应统一格式：

```json
{
  "code": 200,
  "msg": "success",
  "data": {},
  "requestId": "uuid",
  "timestamp": 1770000000
}
```

4. 所有异常响应统一格式：

```json
{
  "code": 400,
  "msg": "参数错误",
  "requestId": "uuid",
  "timestamp": 1770000000,
  "errors": ["字段校验失败"]
}
```

### 2.2 状态码规范

- `200`：成功
- `400`：参数错误
- `401`：未登录、token 无效或登录态失效
- `403`：权限不足
- `404`：资源不存在
- `409`：业务状态冲突
- `500`：服务器内部错误

### 2.3 认证与安全

1. 登录成功返回 `accessToken` 和 `refreshToken`。
2. 前端仅保存登录态 token，不保存明文密码。
3. 登录态失效统一走 `401 -> 刷新 token -> 刷新失败后跳转登录页`。
4. 高危接口必须记录 `requestId`、操作人、角色、IP、对象、审批人和结果。
5. 所有接口均执行 `角色 + 资源 + 操作` 三重校验。

### 2.4 参数约束

1. 用户名长度不超过 20 位，密码至少 8 位，且包含大小写字母、数字和特殊字符。
2. 班级名称不超过 30 位，问题描述不超过 500 字。
3. 导入文件不超过 10MB，图片不超过 2MB，导出文件单个不超过 50MB。
4. 时间格式统一为 `YYYY-MM-DD HH:mm:ss`。
5. 枚举值：
   - `roleType`: `["student","teacher","admin","super_admin","auditor","ops"]`
   - `questionType`: `["single","multi","subjective"]`
   - `examStatus`: `["draft","published","underway","paused","ended","terminated","recycled"]`
   - `issueType`: `["business","exam","system"]`

## 三、全量接口详情

### 3.1 账户服务（Account Service）

接口前缀：`/api/account`
本地联调端口：`8081`（`exam-account`）

| 接口功能 | 请求路径 | 方法 | 请求参数 | 权限校验规则 |
| --- | --- | --- | --- | --- |
| 账户创建 | `/create` | POST | `{roleType, username, phone?, email?, password, realName?}` | 超级管理员可创建全部角色；普通管理员仅可创建教师/学生；学生仅可自助注册自身 |
| 账户查询 | `/query` | POST | `{accountId?, roleType?, keyword?, classId?, pageNum, pageSize}` | 普通管理员仅查教师/学生；教师仅查本班学生；学生仅查自身；审计员仅读取脱敏结果 |
| 账户修改 | `/update` | PUT | `{accountId, username?, avatar?, realName?, phone?, email?}` | 普通管理员不可修改超级管理员；教师仅改自身及本班学生基础信息；学生仅改自身非关键信息 |
| 账户冻结/解冻 | `/freeze` | PUT | `{accountId, isFrozen, reason}` | 普通管理员仅可冻结/解冻教师和学生；超级管理员可处理高权账户 |
| 账户注销 | `/delete` | DELETE | `{accountId}` | 学生仅注销自身；管理员注销其他账户需超级管理员审批 |
| 密码重置 | `/reset-password` | PUT | `{accountId, newPassword, verifyCode}` | 超级管理员和普通管理员可重置教师/学生密码，必须触发通知 |
| 账户审核 | `/audit` | PUT | `{accountId, auditResult, reason?}` | 超级管理员和普通管理员可审核待激活教师/学生账户 |
| 用户登录 | `/login` | POST | `{account, password, verifyCode?}` | 支持用户名、手机号、邮箱登录；冻结账户不可登录 |
| 用户登出 | `/logout` | POST | `{accessToken, refreshToken}` | 登录用户可用，登出后登录态失效 |
| 登录日志查询 | `/login-log/query` | POST | `{accountId?, startTime?, endTime?, pageNum, pageSize}` | 用户仅查自身；审计员和超级管理员可查全局；普通管理员仅查教师/学生 |

| 班级修改 | `/update` | PUT | `{classId, className?, description?, teacherId?}` | 教师仅改自身创建班级；管理员可改全部 |

| 学生自主退班 | `/quit` | PUT | `{classId, studentId}` | 学生仅可退出非强制班级 |
| 班级批量导入 | `/import` | POST | `FormData(file, defaultTeacherId?)` | 仅管理员可导入班级数据 |
| 班级列表导出 | `/export` | GET | `?classId=xxx` | 教师仅导出自身班级；管理员和审计员可导出授权范围数据；导出列已包含 `description`、`approvedMemberCount`、`pendingMemberCount`、`createdBy`、`createTime`、`updateTime` |

#### 3.2.1 班级导入模板兼容说明

- 当前导入支持两种表头：
  - 标准导入头：`className,description,teacherId,forced,status`
  - 导出回灌头：`classId,classCode,className,description,status,teacherId,forced,approvedMemberCount,pendingMemberCount,createdBy,createTime,updateTime`
- 导入字段规则：
  - `className`：必填，空值直接记为跳过并返回错误
  - `description`：可选
  - `teacherId`：可选；为空时回退使用表单参数 `defaultTeacherId`；若存在但不是数字则报错
  - `forced`：可选；为空时默认 `false`；按 `true/false` 解析
  - `status`：可选；为空时默认 `active`
- 导出回灌头中的以下列当前仅用于兼容模板，不参与导入落库：
  - `classId`
  - `classCode`
  - `approvedMemberCount`
  - `pendingMemberCount`
  - `createdBy`
  - `createTime`
  - `updateTime`
- 标准导入样例：

```csv
className,description,teacherId,forced,status
Java 1班,2026春季Java基础班,3001,false,active
Python 提高班,面向竞赛方向,,true,active
```

- 导出回灌样例：

```csv
classId,classCode,className,description,status,teacherId,forced,approvedMemberCount,pendingMemberCount,createdBy,createTime,updateTime
1,CLS00000001,Java 1班,2026春季Java基础班,active,3001,false,32,2,2001,2026-04-03T10:00:00,2026-04-03T10:30:00
```

#### 3.2.2 班级导出字段说明

- 当前导出表头固定为：

```text
classId,classCode,className,description,status,teacherId,forced,approvedMemberCount,pendingMemberCount,createdBy,createTime,updateTime
```

- 字段说明：
  - `classId`：班级主键 ID
  - `classCode`：班级编码，供学生申请入班使用
  - `className`：班级名称
  - `description`：班级描述
  - `status`：班级状态，当前主链路默认 `active`
  - `teacherId`：班级归属教师 ID
  - `forced`：是否强制班级，导出值为 `true/false`
  - `approvedMemberCount`：已通过成员数
  - `pendingMemberCount`：待审批成员数
  - `createdBy`：创建人 ID
  - `createTime`：创建时间
  - `updateTime`：更新时间
- 导出范围规则：
  - 传 `classId` 时仅导出指定班级
  - 不传 `classId` 时导出当前角色可见范围内的全部班级

已实现补充口径：

- `/export`
  - 当前返回 `ApiResponse<QuestionExportView>`，字段为 `{fileKey, fileName, recordCount, masked, generatedAt}`。
  - 当前会把真实 CSV 导出内容写入共享本地资源目录，再由 `/api/resource/file/download?fileKey=...` 统一分发。
  - 审计员导出时 `masked=true`，答案与解析保持脱敏。

| 试卷终止 | `/terminate` | PUT | `{paperId, reason}` | 仅创建者或管理员可终止，且当前仅允许终止 `PUBLISHED` 试卷 |

已实现返回口径：

- `/delete` 返回 `ApiResponse<String>`，`data = "ok"`。
- 试卷状态当前固定为 `DRAFT/APPROVED/REJECTED/PUBLISHED/TERMINATED/RECYCLED`。

首批业务规则补充：

- 手动组卷、自动组卷创建成功后默认进入 `DRAFT`。
- 自动组卷若未传 `examTime`，默认 `90` 分钟；若未传 `passScore`，默认 `ceil(totalScore * 0.6)`。
- 当前按 `1 题 = 1 分` 处理，`totalScore = questionIds.size()`。
- `knowledgeRatio` 当前按 `categoryId` 字符串权重解释，因为题库暂未提供独立知识点字段。
- 教师组卷可用题范围为“本人题目 + 已审核通过的全局题目”，不可使用其他教师的个人题目。
- 发布时教师只能选择自己创建的班级。
- 终止时会把状态切换到 `TERMINATED`，并把终止原因记入 `exam_paper_audit_log`。
- 导出接口当前返回资源元数据，真实 CSV 会先落共享本地资源目录；CSV 内容包含导出人水印，审计员导出必须带审批人 `approverId`。

- 暂无

首批业务规则补充：

- 当前考试创建成功后默认进入 `DRAFT`。
- 当前创建考试要求试卷状态必须为 `PUBLISHED`。
- 当前创建班级必须是该试卷已发布班级范围的子集。
- 教师仅能为自己创建的班级创建考试；管理员可跨班级创建。
- 当前考试查询支持 `{examId?, classId?, studentId?, status?, keyword?, pageNum, pageSize}`。
- 教师查询仅返回本人创建班级下的考试；管理员和审计员可全量只读。
- 学生查询当前按显式分发结果返回本人可见考试，不支持越权指定其他 `studentId`。
- 当前考试参数修改仅支持 `startTime`、`duration` 两个现有字段。
- 未开考前允许调整 `startTime` 与 `duration`；开考后仅允许把 `duration` 调大，不允许改 `startTime`。
- `ENDED`、`TERMINATED` 状态当前不允许再改考试参数。
- 当前考试分发会把 `studentIds` 收口到考试班级内且 `exam_class_member.status=APPROVED` 的学生。
- 当前考试分发采用显式名单全量替换；重新分发后，学生查询结果以最新分发表为准。
- 当前考试暂停仅允许 `PUBLISHED/UNDERWAY -> PAUSED`。
- 当前考试恢复仅允许 `PAUSED -> PUBLISHED/UNDERWAY`，恢复目标按当前时间是否已到 `startTime` 推断。
- 暂停与恢复会把 `operatorId/operatorRole/requestId/fromStatus/toStatus/reason/operateTime` 写入 `exam_instance_status_log`。
- 当前补考申请仅允许学生本人发起，不支持代申请。
- 当前补考申请要求本场考试已结束，且当前学生尚未提交答卷。
- 当前补考审核仅允许处理 `PENDING` 申请；教师只能审核自身班级学生申请，管理员可全量审核。
- 当前补考审核结果仅支持 `APPROVED/REJECTED`；驳回时必须填写原因。
- 当前按 `exam_id + student_id` 唯一约束冻结“单场考试仅 1 次补考机会”的最小落表语义。

### 3.6 考试实时服务（Exam Realtime Service）

接口前缀：`/api/exam/realtime`
本地联调端口：`8090`（`exam-realtime`，已落地）
独立收口文档：`docs/architecture/exam-realtime技术栈与首批落地方案.md`

| 接口或事件 | 触发方式 | 请求参数 | 权限校验规则 |
| --- | --- | --- | --- |
| 会话初始化 | `POST /session` | `{examId}` | 仅允许考生本人在考试窗口期进入；返回考试信息、题目列表、草稿答案和剩余时间 |
| `enterExam` | `WebSocket.emit("enterExam")` | `{examId}` | 仅允许考生本人在考试窗口期进入，禁止多端并发 |
| 自动保存 | `POST /save-progress` | `{examId, answers, currentQId?}` | 仅考生本人；保存草稿到 Redis，不直接写正式成绩表 |
| 切屏上报 | `WebSocket.emit("reportScreen")` | `{examId, screenOutCount}` | 仅考试中可上报；当前仅同步切屏次数，不扩展监考推送 |
| 连接确认 | `emit("connected")` | `{examId, studentId}` | 进入考试房间成功后返回 |
| 倒计时同步 | `emit("countdown")` | `{examId, remainTime}` | 仅推送当前考试中的考生 |
| 试卷提交 | `PUT /submit` | `{examId, answers}` | 由 `exam-realtime` 对外承接，但最终正式落成绩表仍委托 `exam-core` |
| 考试异常申报 | `POST /report-abnormal` | `{examId, type, desc, imgUrls?, screenOutCount?}` | 学生可申报答题异常，教师可申报监考异常；当前直接落最小异常记录表 |

当前已落地口径：

- `/session`
  - 当前请求结构为 `{examId}`，响应结构为 `{exam, questions, answers, currentQId?, screenOutCount}`。
  - `exam` 固定包含 `examId`、`examName`、`status`、`duration`、`remainSeconds`、`startTime`、`endTime`。
  - `questions` 固定包含 `questionId`、`stem`、`type`、`sortNo`、`assignedScore`、`options`。
  - 当前只允许学生调用；考试不存在、未分发、未开始或已正式提交时拒绝进入。
- `/save-progress`
  - 当前请求结构为 `{examId, answers, currentQId?}`，响应结构为 `{saved, savedAt, currentQId?, screenOutCount}`。
  - 当前只写 Redis 草稿，不更新 `exam_score_record/exam_score_detail`。
- `/submit`
  - 当前请求结构为 `{examId, answers}`，响应结构为 `{examId, studentId, status, submittedAt, answeredCount}`。
  - 当前会合并 Redis 草稿与本次提交答案，由 `exam-realtime` 调用 `exam-core /api/exam/core/submit` 完成正式交卷。
  - 当前超时自动交卷也沿用同一提交流程，并通过 Redis 锁避免重复提交。
- `/report-abnormal`
  - 当前请求结构为 `{examId, type, desc, imgUrls?, screenOutCount?}`，响应结构为 `{abnormalId, reportedAt}`。
  - 当前学生、教师可调用；记录直接写入 `exam_realtime_abnormal_record`，不进入 `exam-issue-core` 主流程。
- Socket
  - 当前前端连接时需携带登录 token；服务端握手后解析身份。
  - 当前 `enterExam` 成功后会先下发 `connected`，再按秒推送 `countdown`。
  - 当前同一学生同一考试只保留 1 个活跃连接，新连接会顶掉旧连接。

当前已落地口径：

- `/auto-score`
  - 当前请求结构为 `{examId, studentId}`，返回结构为 `{examId, studentId, objectiveScore, totalScore, status, scoredQuestionCount}`。
  - 当前允许学生触发本人自动评分，教师可触发本人班级学生自动评分，管理员可全局触发；审计员和运维无权限。
  - 当前仅处理 `questionType=single/multi` 的客观题，不扩展判断题、填空题等其他自动判分规则。
  - 当前判分规则按现有字段收口：`single` 题按去首尾空格后忽略大小写精确匹配；`multi` 题按 `, / ， / ; / ； / 、 / 空白` 拆分后排序比对。
  - 当前若成绩记录已是 `PUBLISHED`，则拒绝重新自动评分；重复执行会覆盖客观题得分、正确性与成绩汇总。
  - 当前自动评分后会重算 `objectiveScore/totalScore`；若同一成绩下主观题均已批阅或不存在主观题，则状态推进为 `SCORED`，否则维持 `SCORING`。
- `/query`
  - 当前返回分页结构 `PageResult<ScoreQueryView>`，记录字段最小集为：`scoreId`、`examId`、`examName`、`classId`、`className`、`studentId`、`studentName`、`totalScore`、`objectiveScore`、`subjectiveScore`、`status`、`publishStatus`、`appealId`、`recheckStatus`、`submittedAt`、`publishedAt`。
  - 学生仅能查询本人；教师仅能查询本人创建班级下的成绩；管理员与审计员可全局只读查询。
  - 当前兼容状态字段映射为：`publishStatus = published/unpublished`；`recheckStatus = none/pending/processed`，按该成绩最新一条申诉记录推导。
- `/detail`
  - 当前返回结构 `ScoreDetailView`，除成绩头信息外附带 `questionDetails` 明细列表。
  - 单题明细最小字段集为：`questionId`、`sortNo`、`questionType`、`questionStem`、`studentAnswer`、`correctAnswer`、`assignedScore`、`score`、`correct`、`reviewComment`。
  - 学生仅能查看本人明细；教师仅能查看本人班级；管理员与审计员只读。
- `/manual-score`
  - 当前请求结构为 `{examId, studentId, questionId, score, comment?}`，返回结构为 `{examId, studentId, questionId, score, assignedScore, reviewComment, subjectiveScore, totalScore, status}`。
  - 当前允许教师批阅本人班级学生的主观题，管理员可全局批阅；其他角色无权限。
  - 当前仅允许批阅 `questionType=subjective` 的题目，且 `score` 必须在 `0 ~ assignedScore` 之间。
  - 当前若成绩记录已经处于 `PUBLISHED`，则拒绝继续人工批阅。
  - 当前按现有字段收口：`reviewComment != null` 视为该主观题已批阅；若前端未传评语，服务端会回写空字符串作为“已批阅”标记。
  - 当前每次批阅后都会重算 `subjectiveScore/totalScore`；当同一成绩下全部主观题都已批阅时，状态推进为 `SCORED`，否则维持 `SCORING`。
- `/publish`
  - 当前请求结构为 `{examId, classId}`，返回结构为 `{examId, classId, publishedCount, status, publishedAt}`。
  - 当前允许教师发布本人班级成绩，管理员可全局发布；其他角色无权限。
  - 当前仅允许目标范围内成绩状态为 `SCORED/PUBLISHED` 时执行发布；若存在 `PENDING/SCORING` 记录则整体拒绝。
  - 当前按幂等语义执行：仅把 `SCORED` 记录推进为 `PUBLISHED`，已是 `PUBLISHED` 的记录保持不变。
- `/analyze`
  - 当前请求结构为 `{examId, classId?, dimensions?}`，其中 `dimensions` 仅允许 `overview`、`status`、`scoreRange`、`class`，为空时返回全部维度。
  - 当前返回结构为聚合视图 `ScoreAnalyzeView`，固定头信息字段包含：`examId`、`examName`、`classId`、`passScore`、`totalScore`、`totalParticipants`、`finishedParticipants`、`generatedAt`。
  - 当前 `overview` 返回 `averageScore`、`highestScore`、`lowestScore`、`passCount`、`passRate`；`statusDistribution` 按 `PENDING/SCORING/SCORED/PUBLISHED` 返回；`scoreRangeDistribution` 按 `0-59/60-69/70-79/80-89/90-100` 返回；`classDistribution` 返回 `classId`、`className`、`studentCount`、`finishedCount`、`averageScore`、`passRate`。
  - 当前仅教师、管理员、审计员可分析成绩；教师仅可分析本人创建班级对应成绩，管理员和审计员可全局只读分析。
  - 当前 `overview`、`scoreRangeDistribution`、`classDistribution` 只统计 `SCORED/PUBLISHED` 成绩；`statusDistribution` 统计全部可见成绩。
  - 当前及格率使用 `exam_instance.paper_id -> exam_paper.pass_score` 计算；分数段按 `exam_paper.total_score` 折算百分比后分桶。
- `/apply-recheck`
  - 当前请求结构为 `{examId, studentId, questionId, reason}`，返回结构为 `{appealId, scoreId, questionId, status, recheckStatus, createdAt}`。
  - 当前仅学生本人可发起复核申请；目标成绩必须存在且属于本人，成绩状态必须为 `PUBLISHED`，并且 `publishedAt` 距当前不超过 7 天。
  - 当前仅允许针对 `questionType=subjective` 的题目发起复核；同一 `scoreId + questionId` 已存在 `PENDING` 申诉时拒绝重复提交。
  - 当前申诉会落库到 `exam_score_appeal`，初始状态固定为 `PENDING`，返回中的 `recheckStatus` 固定为 `pending`。
- `/handle-appeal`
  - 当前请求结构为 `{appealId, handleResult, reason}`，其中 `handleResult` 仅允许 `approved/rejected`，返回结构为 `{appealId, scoreId, questionId, status, recheckStatus, handleResult, scoreStatus, handledAt}`。
  - 当前允许教师处理本人班级学生申诉，管理员可全局处理；学生、审计员、运维无权限。
  - 当前仅允许处理 `PENDING` 申诉；处理后申诉状态落为 `APPROVED/REJECTED`，`recheckStatus` 统一转为 `processed`。
  - 当前若处理结果为 `APPROVED` 且对应成绩原状态为 `PUBLISHED`，则成绩状态会回退为 `SCORING` 并清空 `publishedAt`，后续需重新人工批阅并再次发布。
- `/export`
  - 当前请求结构为 `GET /api/score/export?examId=xxx&classId=xxx&includeAnalysis=false`，返回结构为 `{fileKey, fileName, recordCount, includeAnalysis, generatedAt}`。
  - 当前允许教师导出本人班级成绩，管理员和审计员可导出授权范围内成绩；学生、运维无权限。
  - 当前会把真实 CSV 导出内容写入共享本地资源目录，`fileKey` 可直接对接 `/api/resource/file/download`。
  - 当前 `includeAnalysis=true` 时会按相同筛选范围补分析标记，但不会额外改变返回结构。
- `/update`
  - 当前请求结构为 `{scoreId, newScore, reason, approverId}`，返回结构为 `{scoreId, previousTotalScore, totalScore, objectiveScore, subjectiveScore, approverId, updatedAt}`。
  - 当前仅超级管理员可修改成绩；`approverId` 必须与当前操作人不同。由于成绩服务当前未内建账号快照，是否为“另一名超级管理员”由上层权限体系保证。
  - 当前仅允许修改 `SCORED/PUBLISHED` 状态记录；`newScore` 不得小于当前 `objectiveScore`，也不得大于试卷 `totalScore`。
  - 当前修改会直接重算 `subjectiveScore = newScore - objectiveScore`，保持 `objectiveScore` 不变，并同步写入 `exam_score_change_log` 审计留痕。

### 3.8 系统服务（System Service）

接口前缀：`/api/system`
本地联调端口：`8085`（`exam-system`）

| 接口功能 | 请求路径 | 方法 | 请求参数 | 权限校验规则 |
| --- | --- | --- | --- | --- |
| 权限查询 | `/permission/query` | POST | `{roleType?, accountId?}` | 登录用户可查询自身权限清单；管理员可查询授权范围内账户 |
| 角色创建/修改 | `/role` | POST/PUT | `{roleId?, roleName, permissionIds, templateId?}` | 仅超级管理员可操作 |
| 权限分配 | `/permission/assign` | PUT | `{accountId, roleId, expireTime?}` | 普通管理员仅可分配教师/学生相关权限；超级管理员可分配全部 |
| 系统配置查询 | `/config/query` | POST | `{configKey?, category?, pageNum, pageSize}` | 超级管理员可查全部；普通管理员、审计员、运维仅查授权范围 |
| 系统配置修改 | `/config/update` | PUT | `{configKey, configValue, verifyCode}` | 仅超级管理员可修改 |
| 告警配置查询 | `/alarm/query` | POST | `{alarmType?, pageNum, pageSize}` | 超级管理员、审计员、运维可查询 |
| 告警配置修改 | `/alarm/setting` | PUT | `{alarmType, threshold, recipients}` | 仅超级管理员可修改 |
| 系统日志查询 | `/log/query` | POST | `{logType?, operator?, startTime?, endTime?, pageNum, pageSize}` | 普通管理员仅查看业务日志；审计员看全量；运维仅看系统日志 |
| 系统日志导出 | `/log/export` | GET | `?logType=xxx&startTime=xxx&endTime=xxx&approverId=xxx` | 审计员导出需超级管理员审批 |
| 备份记录查询 | `/data/query` | POST | `{backupType?, status?, startTime?, endTime?, pageNum, pageSize}` | 仅超级管理员、审计员、运维可查；支持按时间区间过滤；结果补充 `updateTime`、`canRestore`、`lifecycleStage`；`status` 可取 8 个异步生命周期状态 |
| 数据备份 | `/data/backup` | POST | `{backupType, remark?}` | 仅系统运维执行备份；`backupType` 仅支持 `full/incremental/config/audit`；成功返回任务受理结果 `{backupId, status, lifecycleStage}` |
| 数据恢复 | `/data/restore` | POST | `{backupId, verifyCode1, verifyCode2}` | 仅超级管理员恢复，且需双人授权；仅 `BACKUP_SUCCESS` 或 `RESTORE_FAILED` 状态记录允许进入恢复受理；成功返回任务受理结果 `{backupId, status, lifecycleStage}` |

- `system.data.query.status` 固定枚举：
  - `BACKUP_PENDING`
  - `BACKUP_RUNNING`
  - `BACKUP_SUCCESS`
  - `BACKUP_FAILED`
  - `RESTORE_PENDING`
  - `RESTORE_RUNNING`
  - `RESTORE_SUCCESS`
  - `RESTORE_FAILED`
- `lifecycleStage` 固定映射：
  - `*_PENDING -> accepted`
  - `*_RUNNING -> running`
  - `BACKUP_SUCCESS -> available`
  - `RESTORE_SUCCESS -> completed`
  - `*_FAILED -> failed`
- `canRestore = true` 仅在 `BACKUP_SUCCESS` 与 `RESTORE_FAILED`。
- 配置变更与备份/恢复受理、终态会同步写入库内通知出站记录 `system_notification_record`，当前阶段不直接对外提供通知查询接口。
- `/log/export`
  - 当前返回 `ApiResponse<SystemLogExportView>`，字段为 `{fileKey, fileName, recordCount, generatedAt}`。
  - 当前会把真实日志 CSV 写入共享本地资源目录，再由资源服务统一按 `fileKey` 分发；审计留痕规则保持不变。

### 3.9 问题申报核心服务（Issue Core Service）

接口前缀：`/api/issue/core`
本地联调端口：`8088`（`exam-issue-core`，已落地）

| 接口功能 | 请求路径 | 方法 | 请求参数 | 权限校验规则 |
| --- | --- | --- | --- | --- |
| 问题申报提交 | `/create` | POST | `{type, title, desc, reporterId, examId?, classId?, imgUrls?}` | 所有角色均可提交，必须明确问题类型 |
| 问题查询 | `/query` | POST | `{issueId?, type?, status?, reporterId?, handlerId?, pageNum, pageSize}` | 申请人看自身；处理人看分派问题；审计员看全量只读 |
| 问题处理 | `/handle` | PUT | `{issueId, handlerId, result, solution?}` | 管理员处理业务类；教师处理本班考试类；运维处理系统类 |
| 问题转派 | `/transfer` | PUT | `{issueId, fromHandlerId, toHandlerId, reason}` | 当前处理人可转派相同类型问题，教师仅可转派考试类问题 |
| 问题关闭 | `/close` | PUT | `{issueId, confirmResult, comment?}` | 处理人可关闭，且需申请人确认无异议 |
| 问题进度跟踪 | `/track` | POST | `{issueId}` | 申请人、处理人、审计员可跟踪 |

当前已落地口径：

- `/create`
  - 当前请求结构为 `{type, title, desc, reporterId?, examId?, classId?, imgUrls?}`，其中 `type` 固定为 `BUSINESS/EXAM/SYSTEM`。
  - 当前所有角色均可提交；`reporterId` 以当前登录人上下文为准；`EXAM` 类型至少要求 `examId` 或 `classId` 存在其一。
  - 当前响应返回 `IssueView`，固定包含 `issueId`、`type`、`title`、`description`、`status`、`reporterId`、`currentHandlerId`、`examId`、`classId`、`latestResult`、`latestSolution`、`imgUrls`、`createTime`、`updateTime` 及可解析的名称快照字段。
  - 为避免 Snowflake Long ID 在前端 JavaScript 环境发生精度丢失，当前 `issueId`、`reporterId`、`currentHandlerId`、`examId`、`classId` 统一按字符串返回。
- `/query`
  - 当前请求结构为 `{issueId?, type?, status?, reporterId?, handlerId?, pageNum, pageSize}`，响应为 `PageResult<IssueView>`。
  - 当前管理员、审计员可全局只读查询；其余角色只可查询“本人申报”或“当前分派给本人”的问题。
  - 当前教师被分派考试类问题后，可继续查询、处理、转派、关闭和跟踪该问题，不要求其必须是考试原始创建教师。
- `/handle`
  - 当前管理员仅可处理 `BUSINESS`；教师仅可处理 `EXAM`；运维仅可处理 `SYSTEM`；学生与审计员无权限。
  - 当前 `PENDING` 问题进入处理后状态转为 `PROCESSING`，并回写 `currentHandlerId`、`latestResult`、`latestSolution`。
- `/transfer`
  - 当前仅允许 `PROCESSING` 状态且“当前处理人本人”发起转派。
  - 当前响应继续返回最新 `IssueView`；状态保持 `PROCESSING`，仅更新 `currentHandlerId` 并写入轨迹日志。
- `/close`
  - 当前仅允许 `PROCESSING` 状态且“当前处理人本人”关闭问题。
  - 当前 `confirmResult` 固定要求传 `confirmed`，关闭后状态转为 `CLOSED`。
- `/track`
  - 当前返回 `IssueTrackView`，其中 `logs` 按时间顺序返回 `CREATED/HANDLED/TRANSFERRED/CLOSED` 轨迹。
- 当前申请人、当前处理人、管理员、审计员可跟踪；问题实时推送已由 `exam-issue-notify` 承接，本服务当前只提供主流程与轨迹查询。

### 3.10 问题实时通知服务（Issue Notify Service）

接口前缀：`/api/issue/notify`
本地联调端口：`8091`（`exam-issue-notify`，已落地）
独立收口文档：`docs/architecture/exam-issue-notify技术栈与首批落地方案.md`

| 接口或事件 | 触发方式 | 请求或响应参数 | 说明 |
| --- | --- | --- | --- |
| 通知查询 | `POST /api/issue/notify` | 请求 `{cursor?, limit?}`；响应 `{scope, cursor, notifications}` | 返回当前登录用户可见的最近一批通知 |
| `subscribeIssue` | `WebSocket.emit("subscribeIssue")` | `{scope: "issue", cursor?}` | 建立问题通知订阅 |
| `connected` | `emit("connected")` | `{scope, userId, cursor}` | 订阅成功确认 |
| `issueNotify` | 服务端自动推送 | `{issueId, title, type, status, handlerId, handlerName, reporterId, reporterName, processDesc, action, time}` | 业务类推给管理员，考试类推给本班教师，系统类推给运维；审计员按只读口径可见 |
| `processNotify` | 服务端自动推送 | `{issueId, title, type, status, handlerId, handlerName, reporterId, reporterName, processDesc, action, time}` | 推送给申请人和当前处理人；审计员按只读口径可见 |

当前说明：

- 当前首版直接读取 `exam_issue_record` 与 `exam_issue_process_log`，不新增数据库表，不引入 MQ。
- 当前前端建议单独使用 `VITE_ISSUE_WS_BASE_URL` 直连 `exam-issue-notify`；若未单独配置，则回退到 `VITE_WS_BASE_URL`。
- 当前问题列表与轨迹详情仍由 `/api/issue/core/query` 与 `/api/issue/core/track` 提供，`exam-issue-notify` 只负责实时通知。
- 当前 `issueNotify.issueId` 与 `processNotify.issueId` 统一按字符串返回，避免前端对 Snowflake Long ID 发生精度丢失。

### 3.11 资源服务（Resource Service）

接口前缀：`/api/resource`
本地联调端口：`8089`（`exam-resource`，已建模块）

| 接口功能 | 请求路径 | 方法 | 请求参数 | 权限校验规则 |
| --- | --- | --- | --- | --- |
| 试题图片获取 | `/question/img` | GET | `?imgKey=xxx&token=xxx` | 教师查看自身或全局试题图片；学生仅在考试中查看当前试题图片 |
| 试卷模板下载 | `/paper/template` | GET | `?paperType=single|multi|mix&token=xxx` | 教师和管理员可下载 |
| 导出文件下载 | `/file/download` | GET | `?fileKey=xxx&token=xxx` | 仅文件创建者或授权管理员可下载 |

当前已落地口径：

- `/question/img`
  - 当前请求结构为 `GET /api/resource/question/img?imgKey=xxx&token=xxx`，返回结构为 `{imgKey, fileName, contentType, accessPath, size}`。
  - 当前允许教师、管理员、审计员查看题图；学生也可读取题图元数据以兼容考试中查看当前试题图片；运维无权限。
  - 当前服务按 `imgKey` 在本地资源目录生成或读取占位 SVG 文件，不读取题库真实图片表；`token` 仅作兼容参数保留，当前不参与服务内鉴权。
- `/paper/template`
  - 当前请求结构为 `GET /api/resource/paper/template?paperType=single|multi|mix&token=xxx`，返回结构为 `{paperType, fileName, contentType, downloadPath, size}`。
  - 当前允许教师和管理员下载模板；学生、审计员、运维无权限。
  - 当前模板文件按 `paperType` 在本地资源目录生成 CSV，占位内容覆盖 `single`、`multi`、`mix` 三类固定模板。
- `/file/download`
  - 当前请求结构为 `GET /api/resource/file/download?fileKey=xxx&token=xxx`，返回结构为 `{fileKey, fileName, contentType, downloadPath, size}`。
  - 当前按 `fileKey` 前缀执行最小授权：`question-export/paper-export/score-export` 允许教师、管理员、审计员；`system-log-export` 仅允许管理员、审计员；其余键默认仅管理员可访问。
  - 当前服务只读取共享本地资源目录中已存在的真实导出文件；题库、试卷、成绩、系统日志导出已接入该目录回写链路，暂未接入真实文件登记表或对象存储。

## 四、前后端联调检查项

1. 前端页面中出现的以下接口必须与本文件保持一致：
   - `/api/system/permission/query`
   - `/api/account/audit`
   - `/api/account/login-log/query`
   - `/api/class/query`
   - `/api/class/apply-join`
   - `/api/class/import`
   - `/api/exam/core/query`
   - `/api/score/detail`
   - `/api/system/config/query`
   - `/api/system/data/query`
2. 所有复杂查询接口均为 `POST /query`，不得再使用带请求体的 `GET` 查询。
3. 登录响应统一返回 `accessToken` 和 `refreshToken`，前端异常处理必须纳入 `401`。
4. 非实时业务统一走 HTTPS 网关，实时接口统一走 WebSocket。

## 五、部署与日志规范

1. Spring Boot 服务容器化部署，统一接入 MySQL、Redis 和消息通知组件。
2. Node.js 服务容器化部署，用于考试实时交互和问题通知。
3. 资源服务当前为 Spring Boot 实现，沿用共享本地资源目录分发链路。
4. 所有服务输出统一结构日志，至少记录 `requestId`、操作人、角色、IP、耗时、结果。
5. 敏感操作如成绩修改、数据恢复、日志导出需单独归档并保留不少于 180 天。
